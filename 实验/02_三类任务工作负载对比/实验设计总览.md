# 实验A — 三类任务负载特征对比

## 实验目标

在相同硬件（树莓派5）和相同模型配置（ACT cs=100）下，采集**抓取 / 推倒 / 分类**三类典型操作任务的时序数据，量化三类工作负载在感知—推理—执行各阶段的时间分布差异。

对应论文第3章"3.2 三类典型工作负载特征"和"3.3 感知—推理—执行三阶段划分"。

---

## 三类任务定义

| 任务 | 英文名 | 负载特征 | 已完成 |
|------|--------|----------|--------|
| 抓取 | Grasp（抓取瓶子） | 高精度末端定位，全程视觉引导 | ✅ 已有数据（so101_act_bottle_cs100） |
| 推倒 | Push（推倒积木） | 接触式开环动作，精度需求低 | ⬜ 待采集 |
| 分类 | Sort（颜色分拣） | 多 pick-place 循环，需视觉判别 | ⬜ 待采集 |

## 对比指标

所有指标从 `timing_stats.csv` 直接读取：

| 指标 | 含义 |
|------|------|
| `fps` | 实际帧率 = frames / episode_total_s |
| `obs_pct` | 观测阶段占总时间比例 |
| `inference_pct` | 推理阶段占总时间比例 |
| `action_pct` | 执行阶段占总时间比例 |
| `wait_pct` | 等待阶段占总时间比例（空闲预算） |
| `inference_ms_per_frame` | 每帧平均推理耗时（ms） |
| `obs_ms_per_frame` | 每帧平均观测耗时（ms） |

## 假设与预期

**假设**：三类任务使用相同 ACT 架构，单次推理时间应接近（模型权重不同但结构一致）。差异主要体现在：

1. **抓取**：高精度要求 → wait_pct 低（推理几乎用满时间预算），inference_pct 高
2. **推倒**：动作简单 → 可能通过较大 cs 完成更多 wait，整体 FPS 相似
3. **分类**：多阶段任务 → episode 时长更长，若在一个 episode 内完成两次 pick-place，推理总次数翻倍

**若推理时间相近（均 ~1–2s/次）**，则三类任务的 FPS 和阶段占比应基本一致，说明瓶颈完全由模型决定而非任务特性——这本身也是一个重要结论（"ACT 推理延迟与任务类型无关"）。
