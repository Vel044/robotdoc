# 推倒任务（Push Task）实验设计

## 任务定义

工作台上放置**两个不同外观的瓶子**（如矿泉水瓶 + 可乐瓶），机械臂将其中**指定的一个**推倒，另一个保持直立不动。

```
初始布局（俯视）：

    ┌─────────────────────────┐
    │                         │
    │    [瓶A]      [瓶B]     │  ← 两瓶间距约 15 cm
    │                         │
    └─────────────────────────┘
              ↑
           机械臂
```

**任务说明文字**（dataset.single_task）：`"推倒左侧瓶子，保持右侧瓶子直立"` 或 `"推倒可乐瓶"`

---

## 与抓取任务的负载差异

| 特征 | 抓取（基准，单瓶） | 推倒（两瓶选一） |
|------|----------------|----------------|
| 视觉任务 | 定位单瓶位置 | 区分两瓶 + 定位目标瓶 |
| 末端执行器 | 夹爪精确夹取 | 夹爪半开，侧向接触推压 |
| 动作精度 | 高（需对准瓶口） | 中（只需侧面接触即可） |
| 避障要求 | 无 | 需绕开非目标瓶 |
| 动作轨迹复杂度 | 中（下降抓取上升） | 中（侧向靠近推压后退） |
| episode 时长 | 30 s | 20–25 s |

**负载特征核心差异**：推倒任务引入了**干扰物**（另一个瓶子），模型需要从图像中区分目标与非目标，是对 ACT 视觉判别能力的考验。

---

## 硬件准备

| 物品 | 规格 |
|------|------|
| 瓶子 A | 矿泉水瓶（透明，500ml） |
| 瓶子 B | 可乐瓶 / 深色瓶（外观与A差异明显） |
| 布局固定 | 桌面贴胶带标记两瓶初始位置，每次复位后严格对齐 |
| 任务固定 | 固定推倒同一瓶（如始终推倒瓶A），降低演示数据歧义 |

---

## 数据采集步骤

### 第一步：收集演示数据

```bash
lerobot-record \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \
  --robot.cameras="{cam: {type: opencv, index_or_path: 0}}" \
  --teleop.type=so101_leader \
  --teleop.port=/dev/ttyACM1 \
  --dataset.repo_id=<your_hf_id>/so101_push_bottle \
  --dataset.single_task="推倒左侧矿泉水瓶" \
  --dataset.num_episodes=30 \
  --dataset.episode_time_s=25 \
  --dataset.reset_time_s=20
```

**演示动作要领**：
1. 手臂从正前方接近目标瓶（瓶A）
2. 夹爪半开（不夹紧），用腕部侧面接触瓶身中部
3. 向侧方或前方横推，使瓶A倒下
4. 手臂退回初始位置
5. 瓶B全程保持直立

### 第二步：训练 ACT 模型

```bash
python lerobot/scripts/train.py \
  --policy.type=act \
  --policy.chunk_size=100 \
  --dataset.repo_id=<your_hf_id>/so101_push_bottle \
  --output_dir=outputs/train/so101_act_push_cs100
```

### 第三步：采集计时数据

```bash
lerobot-record \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \
  --robot.cameras="{cam: {type: opencv, index_or_path: 0}}" \
  --policy.path=outputs/train/so101_act_push_cs100/checkpoints/last/pretrained_model \
  --dataset.repo_id=<your_hf_id>/so101_act_push_timing \
  --dataset.single_task="推倒左侧矿泉水瓶" \
  --dataset.num_episodes=10 \
  --dataset.episode_time_s=25 \
  --dataset.reset_time_s=20
```

计时数据自动写入 `~/lerobot/analysis/timing_stats.csv`，model 字段为 `so101_act_push_cs100`。

---

## 成功/失败判定标准

| 结果 | 判定 |
|------|------|
| 瓶A倒下，瓶B直立 | ✅ 成功 |
| 瓶A倒下，瓶B也被碰倒 | ⚠️ 部分成功（可记录，注明） |
| 瓶A未倒下 | ❌ 失败 |
| 手臂碰到瓶B后退开 | ❌ 失败 |

---

## 预期结论

**推理时间**：与抓取任务接近（同为 ACT cs=100），FPS 约 18–19。
**差异来源**：若推理时间真的相近，说明 ACT 的推理时间与场景中物体数量无关，纯由模型结构决定——这本身是一个有意义的实验结论，说明"视觉复杂度不影响推理延迟"。
