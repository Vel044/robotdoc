# 分类任务（Grasp-One-of-Three Task）实验设计

## 任务定义

工作台上放置**三个不同外观的瓶子**，机械臂抓取并取走其中**指定的一个**，另外两个保持原位不动。

```
初始布局（俯视）：

    ┌──────────────────────────────┐
    │                              │
    │   [瓶A]    [瓶B]    [瓶C]   │  ← 三瓶等间距排列，约 12 cm 间距
    │                              │
    └──────────────────────────────┘
                  ↑
               机械臂
```

**任务说明文字**（dataset.single_task）：`"抓取中间的瓶子"` 或 `"抓取可乐瓶"`

---

## 与抓取任务的负载差异

| 特征 | 抓取（基准，单瓶） | 分类（三瓶选一） |
|------|----------------|----------------|
| 视觉任务 | 定位单瓶 | 从三瓶中识别目标瓶 + 定位 |
| 干扰物 | 无 | 2 个非目标瓶 |
| 避障要求 | 无 | 夹爪需在两侧瓶之间穿过，不能碰倒邻瓶 |
| 末端精度 | 高 | 更高（间距更窄，容错空间小） |
| 语义理解 | 不需要 | 需要区分"哪个是目标" |
| episode 时长 | 30 s | 30–40 s（抓取路径更复杂） |
| 任务难度 | 中 | 高（ACT 最难的场景之一） |

**负载特征核心差异**：三瓶选一是对 ACT **视觉-动作条件对应能力**的最强考验——模型不仅要知道怎么抓，还要知道抓哪个，同时规划出不碰倒旁边两瓶的路径。

---

## 硬件准备

| 物品 | 规格 |
|------|------|
| 瓶子 A | 矿泉水瓶（透明） |
| 瓶子 B | 可乐瓶（深色，目标瓶，放中间） |
| 瓶子 C | 果汁瓶（橙色/绿色） |
| 布局固定 | 三瓶等间距，桌面贴标记，每次复位严格对齐 |
| 任务固定 | 固定抓中间的瓶B（外观最易区分），降低训练难度 |
| 放置区域 | 机械臂正前方 20 cm，三瓶横向展开 |

## 数据采集步骤

### 第一步：收集演示数据

```bash
lerobot-record \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \
  --robot.cameras="{cam: {type: opencv, index_or_path: 0}}" \
  --teleop.type=so101_leader \
  --teleop.port=/dev/ttyACM1 \
  --dataset.repo_id=<your_hf_id>/so101_grasp_one_of_three \
  --dataset.single_task="抓取中间的可乐瓶" \
  --dataset.num_episodes=50 \
  --dataset.episode_time_s=40 \
  --dataset.reset_time_s=25
```

**演示动作要领**：
1. 确认三瓶初始位置对齐
2. 手臂从正前方向中间瓶（瓶B）接近
3. 路径需从两侧瓶之间穿过，全程不碰 A、C
4. 夹爪对准瓶B中部抓取
5. 上升后将瓶B移到桌面一侧放下（或直接放到固定收纳区）
6. 手臂退回初始位置（A、C 保持原位）

> 建议演示量 ≥ 50 条。三瓶场景比单瓶难度高，数据需求更大。每条演示后检查 A、C 是否被碰动，若有轻微移位需重新标记位置后才能 reset。

### 第二步：训练 ACT 模型

```bash
python lerobot/scripts/train.py \
  --policy.type=act \
  --policy.chunk_size=100 \
  --dataset.repo_id=<your_hf_id>/so101_grasp_one_of_three \
  --output_dir=outputs/train/so101_act_sort_cs100
```

### 第三步：采集计时数据

```bash
lerobot-record \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \
  --robot.cameras="{cam: {type: opencv, index_or_path: 0}}" \
  --policy.path=outputs/train/so101_act_sort_cs100/checkpoints/last/pretrained_model \
  --dataset.repo_id=<your_hf_id>/so101_act_sort_timing \
  --dataset.single_task="抓取中间的可乐瓶" \
  --dataset.num_episodes=10 \
  --dataset.episode_time_s=40 \
  --dataset.reset_time_s=25
```

计时数据自动写入 `~/lerobot/analysis/timing_stats.csv`，model 字段为 `so101_act_sort_cs100`。

## 成功/失败判定标准

| 结果 | 判定 |
|------|------|
| 抓到瓶B，A/C 原位不动 | ✅ 成功 |
| 抓到瓶B，但 A 或 C 被碰倒 | ⚠️ 部分成功（动作正确但路径粗糙） |
| 抓到 A 或 C（错误目标） | ❌ 失败（视觉判别错误） |
| 未能抓取任何瓶子 | ❌ 失败 |

## 预期结论

**推理时间**：与抓取任务接近（同为 ACT cs=100），FPS 约 18–19。

**三任务对比核心论点**：

| 任务 | 视觉复杂度 | 场景干扰物 | 预期 inference_pct |
|------|----------|----------|-------------------|
| 抓取（单瓶） | 低 | 无 | ~37% |
| 推倒（两瓶选一） | 中 | 1 个 | ~37%（预期相近） |
| 分类（三瓶选一） | 高 | 2 个 | ~37%（预期相近） |

若三者推理时间确实相近，论文结论为：
> **ACT 的推理延迟由模型参数量和序列长度决定，与场景视觉复杂度（物体数量、干扰物）无关。视觉复杂度影响任务成功率和动作质量，但不影响推理吞吐。**

这一结论与"瓶颈在 GEMM 计算而非感知复杂度"的底层解释一致，是论文第4章优化方向分析的重要依据。
