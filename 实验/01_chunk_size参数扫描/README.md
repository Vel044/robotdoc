# 实验B — chunk_size 参数扫描

## 实验目标

验证 ACT 动作分块（action chunking）机制对推理延迟和 FPS 的影响，量化"分块摊薄推理开销"的理论预测。

## 实验配置

| 参数 | 值 |
|------|----|
| 机器人 | SO101 Follower |
| 模型 | so101_act_bottle（抓取瓶子，ACT） |
| 任务 | 抓取瓶子（Grasp Bottle） |
| 每个 cs 跑的 episode 数 | 20 |
| 每个 episode 时长 | 30 s |
| chunk_size 取值 | 1, 3, 5, 10, 20, 50, 100 |
| 硬件 | 树莓派5，ARM Cortex-A76，无 GPU |

## 数据文件

- `timing_stats.csv` — 原始计时数据，每行一个 episode
- `plot_timing.py` — 绘图脚本（`python plot_timing.py`）
- `chart1_time_pct.png` — 各阶段时间占比 vs chunk_size
- `chart2_fps.png` — FPS vs chunk_size
- `chart3_ms_per_frame.png` — 每帧绝对时间（log scale）vs chunk_size

## 核心结论

| chunk_size | 平均 FPS | inference_pct | wait_pct |
|-----------|---------|--------------|---------|
| 1 | ~0.8 | ~99.7% | ~0% |
| 3 | ~2.1 | ~95% | ~4% |
| 5 | ~3.3 | ~91% | ~8% |
| 10 | ~5.7–6.0 | ~82–88% | ~8–15% |
| 20 | ~6.8–7.3 | ~72–79% | ~14–21% |
| 50 | ~15.1 | ~48% | ~36% |
| 100 | ~18.4–19.5 | ~34–44% | ~39–48% |

**关键观察**：

1. **cs=1 时推理占 99.7%**，系统完全被推理阻塞，FPS 仅 0.8，wait_pct ≈ 0（无空闲时间）。
2. **cs=100 时 FPS≈18–19**，推理占 ~38%，wait_pct ~42%，说明还有剩余时间预算，但远未达到 30 fps 目标。
3. **FPS 与 cs 非线性**：cs 从 1→10 增益巨大（0.8→6），cs 从 50→100 增益趋缓（15→19），符合边际递减规律。
4. **推理绝对时间接近恒定**（~1 次推理 ≈ 1–1.5s），分块是"分摊"而非"减少"推理开销。
5. cs=100 时每次推理覆盖 100 帧 × 33ms = 3.3 s 的动作窗口，与实测推理耗时 ~1s 匹配（提前推理完毕，有 wait）。

## 对论文的支撑

- 证明 **"分块摊薄推理"是 ACT 在 CPU 上可用的核心机制**
- 为第3章"感知—推理—执行时间预算"提供实测数据
- 支撑"扩大 chunk_size 是最直接的无损优化"论点
